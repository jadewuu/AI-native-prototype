# PM Zero-Friction Onboarding Design

## Goal

PM opens Claude Code, runs one command, and can immediately start generating UI pages with AI.

Current friction: no runnable app, dev-facing README, branded content mixed with generic templates, manual scaffold setup.

Target: `git clone <repo> && cd design-system-starter && bash setup.sh` → interactive setup → ready.

## Architecture

```
design-system-starter/
├── setup.sh                    # Entry point: env check + outputs setup prompt
├── project.config.sh           # PM config (empty by default → triggers Q&A)
├── README.md                   # PM-facing, 3-step quick start
├── .gitignore
│
├── templates/
│   ├── SKILL.template.md       # Design spec template with {{PLACEHOLDERS}}
│   ├── CLAUDE.template.md      # AI behavior template, platform-aware
│   └── project.config.sh       # Config template (copied to root on setup)
│
├── patterns/                   # Page templates (platform-agnostic)
│   ├── list-page.md
│   ├── dashboard.md
│   ├── form-page.md
│   └── detail-page.md
│
├── prompts/
│   └── pm-prompt-template.md   # PM prompt templates (platform-agnostic)
│
├── registry/
│   └── components.md           # Business component registry
│
└── references/                 # Generic, unbranded examples
    ├── list-page-example.tsx
    └── dashboard-example.tsx
```

Generated by setup (NOT in repo):
- `CLAUDE.md` — from templates/CLAUDE.template.md
- `SKILL.md` — from templates/SKILL.template.md
- `.mcp.json` — platform-specific MCP config
- `web/` or `mobile/` — scaffold directory

## setup.sh Flow

```
1. Check Node.js >= 22, exit with install guide if missing
2. Copy templates/project.config.sh → project.config.sh (if not exists)
3. Output setup prompt for Claude Code to consume
```

setup.sh does NOT launch a new Claude Code — the PM is already in Claude Code. It only checks the environment and outputs a prompt. Claude Code reads the prompt and executes the setup.

## Claude Code Setup Prompt Steps

1. Read project.config.sh — if empty, enter interactive Q&A:
   - Q1: Project name
   - Q2: Platform (web → Next.js+shadcn, mobile → Expo+heroui-native)
   - Q3: Theme source (default / Figma API / custom input)
2. Acquire Design Tokens per theme choice:
   - default: use built-in tokens (#165dff primary, Nunito Sans, etc.)
   - figma: prompt for Figma file URL + PAT, call REST API, map to token format
   - custom: interactive collection of primary color, font family
3. Generate CLAUDE.md + SKILL.md from templates with variable substitution
4. Generate .mcp.json (shadcn for web, heroui-native for mobile)
5. Scaffold platform directory:
   - web: `npx create-next-app web` + `npx shadcn@latest init`
   - mobile: `npx create-expo-app mobile` + heroui-native install
6. `npm install` + `npm run dev`
7. Report: "Ready! Describe your page and I'll generate it."

## Theme Sources

### Default
Use predefined tokens: primary #165dff, font Nunito Sans. Zero config, fastest path.

### From Figma
PM provides Figma file URL + Personal Access Token.
Claude calls `GET /v1/files/{file_key}/styles` and `GET /v1/files/{file_key}`.
Maps Figma Color/Text styles to SKILL.md token table format.
Missing tokens filled with defaults.

### Custom
Claude interactively asks for: primary color, font family.
Derived tokens (success/danger/text/background/border) auto-computed from primary.

## Success Criteria

- PM runs `git clone && bash setup.sh` in Claude Code without errors
- Interactive Q&A completes and generates valid CLAUDE.md + SKILL.md
- Dev server starts and serves a blank scaffold page
- PM can immediately describe a page and AI generates valid code
